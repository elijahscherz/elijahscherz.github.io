Jira Duplicator

This particular project was done during my time at RightBrain Networks. It started out as a Python script meant to copy simple issues between client boards and company boards. It evolved into its own custom module and class, which leveraged jira-python to access the Jira REST API and pull information from passed in issues. Using an argument system, the program could do multiple issues at once, do a query based on Jira's JQL syntax, or even sync two issues to match one another. I ended up supporting copying comments, priorities, assignees, and subtasks, all depending on the availability of the source and target issues.

Challenges
The most difficult part of building this was dealing with the Jira API which would respond with different amounts of information from issues depending on a variety of variables, including who you were logged in as, what type of project you were getting issues from, etc. The solution to this was using checks to see if a field was available on the issue or subtask, and also seeing if required fields were all available on the source issue.

Personal Projects

A Night to Remember

I've been working on this one for almost a year and a half at this point. My dad was a huge fan of text-based Infocom games (like Zork) when he was a kid and challenged me to make one. After playing one myself, I got a bit inspired and started working on it in the summer of 2018. I first decided to approach the problem using "vanilla" Python. I played a few more of the games and determined which parts of the system I wanted to incorporate. There are multiple ways to interact with the world based on commands and keywords. You can also move in different directions and collect items. While my initial system wasn't as complicated as Infocom's, which could support phrases like "stand on shoulders and leave hole", you could get/check items, interact with certain things like doors with those items, and move.

At this point, I started to get more interested in where I could go with this. I've always enjoyed reading and wanted to make this into a story. I started writing a murder mystery where you need to explore and discover what happened. I decided I wanted to move this project into C# and Unity which would give me easier access to graphics, sounds, and more. Currently, we are working on completing more art, getting some final endings implemented, and brushing up on the details of the story. It's a work in progress, but a fun one.

Discord Chat Bot

This small project, which is somewhat of a constant work in progress, is a way to learn a bit more about how to we make bots act in a human-like fashion. I picked up the Discord API and started learning about what it can do when I built the foundation of a bot in Node.js. Right now, it doesn't do much, but it uses VADER sentiment analysis to post reactions to certain messages and reacts to other messages based on keywords. I polled friends and those online to see what types of words they typically react to and in what way, and then set up the bot to randomly react to messages.

I wanted to incorporate more talking ability into the system, but didn't have too much experience with NLP up to this point. I found that Google's Dialogflow had a Node.js SDK, and I got to work hooking up a very basic chat / response system. There is a lot more I can do with it if I take the time to incorporate more responses In the future, using the Intents functionality that Google provides. I also can just let it loose in a couple of servers and use the responses to train it using Google's built in functionality. I'll make it more complex at some point, but it was just a fun little project.

Discord Soundboard Bot

After building one Discord bot, a friend of mine messaged me and wanted me to make something for him: a bot which could join voice chat and play clips. Using the node-opus module, I was able to get the bot to stream clips. Ask requested, I made a method for tagging these clips in the bot, so that you can then search them while in Discord. The bot will direct message you with a list of relevant clips. If you use one of the bots keywords while chatting, it will start playing it's clips for a variety of hilarious outcomes and ridiculous interruptions. We later on extended this functionality to be able to play music!
